# 주의·속도 축 게임 2종 — 구현 스펙

> 대상: `index.html`(= `brain_app.html`) 단일 HTML SPA "말랑말랑 두뇌체조"
> 목적: 기억 축([[기억축-게임-스펙]]) 다음으로 비어 있던 **주의·속도 축**을 채움.
> 원작 근거: NDS 두뇌 트레이닝의 *색 판별(스트룹)* / *순서 잇기(Trail Making)*.
> 원칙: 서버 없음 · 단일 파일 · 60초 타임어택(속도 축) · 기존 컴포넌트/패턴 재사용.

## 1. 색깔 맞추기 (Stroop) — `id: stroop`

### 게임성
색 이름 단어(예: "빨강")가 **다른 잉크색**으로 표시된다. 플레이어는 **글자의 뜻이 아니라
칠해진 잉크색**에 해당하는 색 버튼을 누른다. → 읽기 자동화 억제(선택적 주의).

### 규칙
- 팔레트: `red/blue/green/yellow/purple/orange` 중 난이도별 앞에서 N색 사용.
- 매 시행: 잉크색 = 팔레트 무작위. 글자 = 확률 `incong`으로 잉크색과 **다른** 색 이름(부조화), 아니면 같은 색(조화).
- 색 버튼(팔레트 색상 칩, 이름은 언어별)은 게임 내내 고정. 잉크색 버튼을 누르면 정답.
- 정답: `score += 10 + (combo-1)*2`, combo↑. 오답: combo=0, 시간 −2초, 버튼 빨간 테두리 플래시.
- 난이도:

  | 난이도 | 색 수 | 부조화 확률 |
  |---|---|---|
  | 🌱 easy | 4 | 0.5 |
  | ⚡ normal | 5 | 0.7 |
  | 🔥 hard | 6 | 0.85 |

- 결과: 최종 점수 · 정답 수 · **최고 콤보** · 최고 기록.
- 상태 객체 `ST`. 핵심 함수: `setStroopDiff` `startStroop` `stNextTrial` `stAnswer` `finishStroop`.
- **주의**: 스트룹 효과는 언어 내 간섭이라 색 이름을 **현재 UI 언어**로 표시(`color.*` 키).

## 2. 순서 잇기 (Number Trail) — `id: trail`

### 게임성
정사각 보드에 숫자 `1..N`이 무작위로 흩어져 있다. **1부터 순서대로** 최대한 빠르게 탭.
판을 다 지우면 다음 판은 `N+1`. 60초 동안 지운 판 수로 경쟁. → 시각 탐색 + 처리 속도.

### 규칙
- 시작 N: easy 5 / normal 7 / hard 10. 판 클리어마다 `score += N`, `N++`.
- 배치: `cols=⌈√N⌉` 격자 셀을 섞어 N개 선택, 각 셀 중심 ±지터. 좌표 8~92%로 클램프(겹침 최소).
- 노드는 원형 버튼(절대배치, `translate(-50%,-50%)`). 맞으면 초록 `done` 고정, 틀리면 흔들림 + 시간 −2초.
- 결과: 최종 점수 · **지운 판 수** · 최고 기록.
- 상태 객체 `TR`. 핵심 함수: `setTrailDiff` `startTrail` `trGenPositions` `trNewBoard` `trTapNode` `finishTrail`.

## 3. 인지 영역 커버리지
- 스트룹 = 선택적 주의 / 반응 억제(inhibition).
- 순서 잇기 = 시각 탐색 / 처리 속도(processing speed) + 순차 주의.
→ 기억 축에 이어 **주의·속도 축**을 채움. 남은 축: 바이럴(MBTI+공유), 유연성(전환과제).
