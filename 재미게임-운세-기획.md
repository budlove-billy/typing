# 재미(cat.fun) 게임 기획 — 말로우 운세 (오늘의 운세 + 라이트 사주)

> 목적: 두뇌게임이 아닌 **재미·바이럴·데일리 재방문**용 게임. 기록/능력치/미션 제외.
> 배치: **moamoa 방식** — `playmallow.com/unse/` 별도 정적 페이지. HOME_GAMES에 external 카드로만 노출.

## 0. 기술 확인 (moamoa 선례)
- moamoa = `/moamoa/index.html` 독립 정적 페이지(자체 header/모달·KST 데일리·localStorage·이모지 공유). HOME_GAMES:
  `{id:'moamoa', grp:'cat.daily', koOnly:true, external:'moamoa/'}` → 카드 클릭 시 `location.href`. 기록/능력치/미션/추천에서 전부 제외.
- **동일 패턴으로 `/unse/` 신설 가능.** Vercel은 저장소의 정적 폴더를 그대로 서빙(별도 설정 불필요). sitemap에 추가.

## 1. 왜 사주팔자(정통) 대신 "라이트 운세"인가
- **정통 사주팔자**는 만세력(음↔양 변환·24절기 입춘 경계·60갑자·천간지지·십성·대운) 계산이 무겁고 오차 민감 → 앱 재미 대비 구현비용 과다.
- **라이트 운세**: 생년월일(+선택 시각)로 **띠·오행**만 뽑고, **오늘 날짜 시드**로 데일리 운세를 템플릿에서 결정 → 가볍고 데일리 재방문 강함, moamoa처럼 매일 새 콘텐츠(자동).
- 포지셔닝: **"재미로 보는 오늘의 운세 🔮"** — 미신/단정 아님 명시(FTC 톤). braintype(유형 테스트)·moamoa(데일리 낱말)와 겹치지 않는 **운세 축**.

## 2. 코어 플로우
1. **입력**: 생년월일(필수) + 태어난 시각(선택). localStorage 저장 → 다음 방문 자동.
2. **타고난(1회성, 생일 기반)**:
   - **띠(12지)**: `(year-4)%12` → 쥐·소·범…돼지 (양력 연도 기준, "정통 입춘 기준 아님" 각주).
   - **오행(五行)**: 천간 `year%10` → 갑을=목/병정=화/무기=토/경신=금/임계=수. 색·성향 연결.
   - **타고난 기질 한 줄**: 띠×오행 템플릿(12×5=60 조합, 실제론 띠 12문구 + 오행 형용사 블렌드).
3. **오늘의 운세(데일리, 생일+오늘날짜 시드)**:
   - **총운 별점** ★1~5, 한 줄 총평.
   - **분야별**: 애정 💗 / 금전 💰 / 건강 🍀 / 학업·일 📚 — 각 짧은 코멘트 + 점수.
   - **행운 3종**: 행운의 색 / 숫자 / 아이템(또는 방향).
   - **오늘의 한마디**(말로우 응원 톤).
   - 시드 = hash(생년월일 + KST yyyy-mm-dd) → 배열에서 결정적 선택(같은 사람=하루 종일 같은 결과, 매일 바뀜).
4. **공유**: **Mallow 운세 카드**(canvas, braintype 공유카드 재사용 패턴) — 띠 이모지+별점+총평+행운색, 하단 playmallow.com 워터마크. Web Share/PNG.
5. **재방문 훅**: "내일 또 봐요" + (선택) 홈 카드에 오늘 미확인 ● 뱃지(moamoa dot 패턴).

## 3. 콘텐츠(내가 작성)
- 띠 12 기질 문구 + 오행 5 형용사.
- 총평 템플릿 ~30, 분야별 코멘트 각 ~20, 행운 색/숫자/아이템 풀, 오늘의 한마디 ~30.
- 전부 한국어, 재미·긍정 톤(부정·공포·의료·투자 단정 금지). **음력/시각 미입력도 동작**(시각은 시주 뉘앙스만 가미).
- **면책 문구**: "본 운세는 재미로 즐기는 콘텐츠이며 어떤 결정의 근거가 될 수 없어요."

## 4. 기술 스펙 (정적 페이지)
- `/unse/index.html` 단일 파일: 자체 header(Mallow 로고·뒤로)·입력 폼·결과 카드·공유. GA4·canonical.
- KST 날짜: `new Date(Date.now()+9*3600*1000).toISOString().slice(0,10)`.
- localStorage: `unse_birth`(생일), `unse_seen`(마지막 확인 날짜, 뱃지용).
- 결정적 시드 해시(문자열→32bit) + 시드 PRNG(mulberry32류)로 배열 인덱싱.
- 접근성/브랜드: 기존 파스텔 팔레트·Mallow 톤 재사용(마스코트 SVG는 페이지에 인라인 복제).
- **koOnly**: 운세=한국 문화 → ko 전용(en/th 카드 숨김, moamoa와 동일). 추후 en 확장은 별도 판단.

## 5. HOME_GAMES 연동(본체)
- `{id:'unse', emoji:'🔮', key:'nav.unse', grp:'cat.fun', koOnly:true, external:'unse/'}` 추가(braintype 옆).
- i18n `nav.unse`(ko만 실사용). 기록/능력치/미션/추천에서 자동 제외(external 처리 기존 로직).
- sitemap.xml에 `/unse/` 추가. 홈/게임 탭 ko에서 카드 노출.

## 6. 대안(채택 시 알려주세요)
- **A. 캐주얼 미니게임**(비두뇌): 예) 타로 한 장 뽑기, 행운 룰렛, 하루 뽑기(가챠) — 더 가볍고 순수 재미. 단 데일리 훅은 운세가 강함.
- **B. 정통 사주 라이트**: 띠·오행에 **일주(日柱) 60갑자**까지(만세력 표 내장) — 정확도↑·콘텐츠비용↑.
  현재 기획은 **연주 기반 라이트**(A/B 중간). 원하면 일주까지 확장 가능.

## 7. 열린 결정
1. 컨셉: **라이트 운세(오늘의 운세+간단 사주)**로 확정? 아니면 캐주얼 미니게임(타로/룰렛)?
2. 경로/이름: `/unse/`(운세) vs `/saju/`(사주) vs `/fortune/`. 기본 제안 `/unse/`, ko 전용.
3. 정확도 범위: 연주 기반 라이트(권장) vs 일주까지(만세력 내장).
4. 착수: 확정 시 정적 페이지 + 콘텐츠(내가 작성) 구현 → 배포.
