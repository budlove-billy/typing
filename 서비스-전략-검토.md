# 말랑말랑 두뇌체조 — 서비스 전략 검토 (기획·마케팅)

> 검토일: 2026-07-09. 대상: 사용자가 제시한 6개 전략 옵션.
> 전제(현재 자산): 서버 없는 단일 HTML(15종 게임+미션+능력치), ko/en/th 3언어, 계정·DB 없음(localStorage),
> 원가 = 도메인+호스팅 수준(정적 파일), 팀 = 1인 + AI.

---

## 0. 한 줄 결론

**"1번(웹 오픈)을 즉시, 2번은 PWA로 흡수, 4·5번은 서버 없이 되는 것부터, 3번은 트래픽 신호 후, 6번(피벗)은 반대 — 단 별도 실험은 가능."**

핵심 논리: 지금 제품은 **출시 비용이 거의 0**이다. 전략 고민보다 **실사용 데이터 확보가 먼저**다.
사용자·리텐션 데이터 없이 3~6번에 투자하는 것은 전부 추측 위에 짓는 것.

## 1. 옵션별 평가

### ① 현 상태로 웹 서비스 오픈 (SEO + 도메인) — ★★★★★ 즉시 실행
- **비용**: 도메인 1~2만원/년 + 무료 정적 호스팅(GitHub Pages/Cloudflare Pages). 사실상 0.
- **효과**: 실사용 데이터(리텐션·완주율·공유율)가 생김 → 이후 모든 의사결정의 근거.
- **해야 할 것(체크리스트)**:
  - [ ] 도메인: 한국 타깃이면 **.com 브랜드형** (예: mallangbrain.com / brainmallang.com). 짧고 한글 발음 그대로.
  - [ ] SEO 기본: `<title>`·meta description·OG 태그(공유 미리보기)·favicon·`lang` 정리, sitemap.xml, robots.txt.
  - [ ] **SPA 약점 보완**: 단일 HTML이라 검색엔진에 잡힐 "페이지"가 1개뿐. 게임별 소개용 정적 랜딩(또는 최소한 홈에 게임별 텍스트 설명 섹션) 필요. 콘텐츠 텍스트가 SEO의 전부다.
  - [ ] **구글**: Search Console 등록 + GA4(또는 경량 애널리틱스).
  - [ ] **네이버**: Search Advisor 등록(사이트 소유 확인) + 네이버는 자체 콘텐츠 생태계 우대 → **블로그/카페 시딩**(두뇌게임 추천 글, 두뇌 유형 테스트 공유)이 검색 유입보다 효과적일 수 있음.
  - [ ] 두뇌 유형 테스트 = 유입 후크. 결과 공유카드에 **URL 워터마크** 필수(현재 'mallang-brain' → 실제 도메인으로).
- **리스크**: 낮음. 유일한 주의는 효능 문구(이미 재미 톤으로 정리됨 — FTC 교훈 유지).

### ② 모바일 버전 준비 후 동시 오픈 — ★★☆☆☆ 별도 개발 반대, PWA로 흡수
- 이미 모바일웹 반응형 완료(하단 탭바·터치 최적화). "앱"이 필요한 이유는 (a)홈화면 아이콘 (b)스토어 노출인데:
  - (a)는 **PWA**(manifest.json + service worker)로 하루 만에 해결 — "홈 화면에 추가" + 오프라인 동작.
  - (b) 스토어 노출은 심사·수수료·업데이트 부담 대비 초기 효과 낮음. 트래픽 검증 후 TWA(안드로이드 래핑)로 충분.
- **동시 오픈을 위해 웹 출시를 늦추는 것은 최악의 선택** — 얻는 것 없이 데이터 확보만 늦어짐.

### ③ 글로벌 오픈 (.com / .ai) — ★★★☆☆ 준비는 되어 있으나 순서가 뒤
- 콘텐츠는 이미 en/th 지원이라 기술 준비는 됨. 그러나 글로벌은 **마케팅 비용이 본체**(경쟁: Lumosity·Peak·Impulse, 광고비 큰 시장).
- **.ai 도메인 반대**: 우리는 AI 제품이 아님. .ai는 비싸고(연 10만원+) 포지셔닝 오해 유발. **.com 하나로 통합**하고 언어는 경로/파라미터로.
- 현실적 글로벌 전략: 태국어를 이미 지원하는 점이 특이 자산 → **동남아(태국) 저경쟁 니치**부터 테스트하는 게 미·유럽 정면승부보다 낫다.

### ④ 게임 보강 + 달성 과정 튜닝 + 사용자간 경쟁 — ★★★★☆ 단, 서버 없이 되는 것부터
- 달성 루프(미션·streak·능력치·공유카드)는 이미 v1 완성. 다음 튜닝은 **데이터 보고** 해야 함(어디서 이탈하는지).
- **경쟁 요소의 함정**: 진짜 랭킹 = 서버+계정+어뷰징 방어 = 운영 비용 발생. 트래픽 없는 상태에선 과투자.
- **서버 없이 가능한 경쟁(추천, 먼저)**:
  - **도전장 링크**: 내 점수를 URL 파라미터로 인코딩 → "이 점수 깨봐" 링크/카드 공유 → 받은 사람 화면에 상대 점수 표시. 서버 0원으로 1:1 경쟁 성립. 바이럴 루프도 됨.
  - **오늘의 미션이 이미 날짜 시드**(모두 같은 세트) → "오늘 미션 점수 인증" 문화 유도 가능.
- 서버 랭킹은 **DAU 신호가 보이면** 그때(Supabase/Firebase 무료 티어로 시작).

### ⑤ 경쟁 서비스 대비 차별화 (FUN 난이도) — ★★★★☆ 방향 동의, 차별점은 이미 있다
- Lumosity류 대비 우리의 실제 차별점(이걸 전면에 내세워야 함):
  1. **가입 없음·설치 없음·무료** — 링크 열면 3초 안에 게임 시작 (경쟁사는 전부 계정+구독 장벽).
  2. **"머리 식힘" 포지셔닝** — 자극적 게임의 반대편, 훈련 강박도 아님. "쉬는 시간에 가볍게 3판" (효능 과장 없는 정직함이 오히려 신뢰 차별점).
  3. **한국어 최적화 + 태국어** — 글로벌 서비스들이 약한 로컬라이징.
  4. **공유카드/유형테스트** — K-바이럴 문법(MBTI식)에 맞는 유입 장치.
- FUN 튜닝 아이디어(저비용 순): 게임 오버 시 "한 판 더" 원탭, 신기록 임박 알림("최고기록까지 20점!"), 미션 완료 연출 강화, 주간 베스트 요약 카드.

### ⑥ 강점이 없으니 AI/개발 툴(이미지·영상 변환)로 피벗 — ★★☆☆☆ 피벗 반대, 분리 실험은 가능
- "강점이 없다"는 진단에 일부 동의(두뇌게임 시장은 레드오션, 기술 해자 없음). 그러나:
  - **변환 툴 시장도 똑같이 레드오션**(무료 툴 수백 개, SEO 상위는 대형 사이트가 점유). 강점 문제는 해결 안 되고 시장만 바뀜.
  - **영상 변환은 서버 비용이 실물**로 발생(트랜스코딩 = CPU/대역폭) — 현재 0원 구조의 장점을 버리는 선택.
  - 지금까지 만든 자산(15게임+3언어+루프)을 버리는 매몰비용.
- **대안**: 피벗이 아니라 **포트폴리오 분리**. 두뇌체조는 그대로 운영(트래픽·브랜드 자산)하고, 툴은
  **브라우저 안에서 끝나는 클라이언트 사이드 툴**(canvas/WebAssembly 이미지 변환·리사이즈 등, 서버 0원)로
  **별도 도메인·별도 실험**. SEO 검색 수요("png jpg 변환")가 명확해 트래픽 실험으로는 유효.
  둘을 한 브랜드에 섞으면 정체성만 흐려짐.

## 2. 권장 로드맵 (게이트 방식 — 데이터가 다음 단계를 연다)

### Phase A. 지금~2주: 출시 (옵션① + ②의 PWA만)
1. 도메인 구매(.com) → Cloudflare Pages/GitHub Pages 배포
2. SEO 패키지: meta/OG/favicon/sitemap/robots + 게임 소개 텍스트 섹션
3. PWA(manifest+SW) → "홈 화면에 추가"
4. Search Console + Naver Search Advisor + GA4
5. 공유카드 워터마크 → 실제 도메인
6. 시딩: 네이버 블로그/카페 2~3곳, 지인 채널 (두뇌 유형 테스트 링크 중심)

**측정**: D1/D7 리텐션, 미션 완주율, 게임별 플레이 수, 공유카드 생성 수, 유입 채널.

### Phase B. 3~8주: 루프 강화 (옵션④⑤의 무서버 부분)
- 도전장 링크(점수 인코딩 1:1 경쟁), "한 판 더" 루프, 신기록 임박 연출
- 데이터 기반 튜닝: 이탈 지점 게임 개선, 인기 게임 전진 배치
- **게이트: D7 리텐션 >10% 또는 주간 공유 >50건 → Phase C 진행. 미달 시 제품 루프 재점검(마케팅 증액 금지)**

### Phase C. 신호 확인 후: 확장 (옵션④ 서버 + ③)
- Supabase/Firebase 무료 티어: 닉네임 랭킹(주간 리더보드, 오늘의 미션 랭킹부터)
- 글로벌: .com 그대로 en 마케팅 테스트, 태국 니치 시딩(이미 th 지원)
- 수익화 검토 시작: 경량 광고(AdSense) 또는 쿠팡 파트너스 — 리텐션 훼손 없는 선에서

### 옵션⑥은 별도 트랙
- 원하면 **클라이언트 사이드 이미지 툴**을 별도 도메인으로 1~2주 실험(서버 0원 유지).
- 두뇌체조 브랜드와 절대 혼합하지 않음.

## 4. 결론 (직설)
- **최대 리스크는 "출시 안 하고 계속 만드는 것"**. 1번을 이번 주에 실행할 수 있고, 해야 한다.
- 2번은 PWA로 흡수(동시 오픈 대기 반대), 3번은 게이트 뒤로, 6번 피벗은 반대(분리 실험만).
- 우리 강점은 기술이 아니라 **"가입 없이 3초, 부담 없는 머리 식힘 + K-바이럴 공유"**라는 포지션이다. 이걸 카피와 랜딩에 그대로 써라.

## 5. 추가 검토 (2026-07-10) — 외부 검토안 + 브랜딩

### 5-1. 외부 검토안(웹 우선 출시론)에 대한 평가: **동의, 본 문서 §2와 동일 결론**
- "웹 우선 → 반응 확인 → PWA/앱화", "동시 앱 준비 반대", "효능 과장 금지", "글로벌은 웹이 실험 유리" — 모두 §1~2와 일치. 독립된 두 검토가 같은 결론 = 방향 신뢰도 높음.
- 검토안의 체크리스트 대비 현재 제품 상태:
  - 3분 내 이해되는 게임 ✅(전 게임 원룰) / 난이도 상승 체감 ✅(적응형·판 확대) / 기록 재방문 훅 ✅(최고기록·미션·streak·능력치)
  - 신뢰감 ✅(FTC-안전 문구, 난이도 가중 IQ) / 모바일 브라우저 편의 ✅(반응형+하단탭)
  - "매일 다시 하게 되는 게임 3개" = **오늘의 미션이 이미 구현**.
- 글로벌 최소 구조 대비: 설정 기반 언어 ✅ / 룰 텍스트 분리(I18N) ✅ / 무로그인 ✅ / 로컬 기록 ✅.
  미비: **URL 기반 언어(?lang=)**, 점수·시간 표기 통일 — Phase A에서 소규모 보강.

### 5-2. 브랜딩 제안(Mallow) 평가: **채택 권장, 단 검증 결과 반영**
- 장점 동의: 짧고, 글로벌 발음 쉬움, marshmallow=말랑 감성 연결, "Mallow — 말랑말랑 두뇌게임" 구조 깔끔.
- 리스크: mallow.com/mallow.app은 선점 → 브랜드 검색 SEO에서 동명 서비스와 경쟁. 상표는 구매 전 별도 확인 필수(KIPRIS/USPTO + 앱스토어 검색).
- **도메인 DNS 실사(2026-07-10, nslookup 기준·등록 가능성은 구매 시 재확인)**:
  | 후보 | 상태 |
  |---|---|
  | **playmallow.com** | **DNS 없음 → 구매 가능성 높음 ★1순위** |
  | mallowgames.com / mallowbrain.com / getmallow.com / mallow.fun / mallow.com / mallow.app | 선점(사용 중) |
  | playmallang.com / mallangbrain.com / mallanggames.com / mallow.games | DNS 없음 → 가능성 있음(예비) |
- **권고**: 브랜드 `Mallow`, 한국어 표기는 `말랑`(말로우보다 기존 자산 연속성), 서브타이틀 `말랑말랑 두뇌게임`,
  도메인 **playmallow.com** 즉시 확보(+여유 되면 mallow.games 리다이렉트용). 확보 후 리브랜딩 범위:
  `<title>`/로고/OG/공유카드 워터마크('mallang-brain'→'playmallow.com')/manifest.

### 5-3. 배포 결정(2026-07-10 확정)
- **등록기관: 가비아** (자동연장 ON 필수급 / WHOIS 정보보호는 **선택** — 무료면 ON, 유료면 스킵 가능(스팸 증가 정도, 추후 추가 가능)). 등록기관↔호스팅은 독립이라 제약 없음.
- **호스팅: Vercel 유지**, Cloudflare Pages는 추후 전환. 정적 파일이라 전환 = repo 연결+DNS 변경(코드 수정 0).
- ⚠️ **Vercel Hobby는 비상업 용도 약관** → 광고/수익화 시작 시점 = Pro 결제 또는 Cloudflare Pages 이전 시점.
- DNS(가비아→Vercel): **네임서버 위임 방식으로 확정**(ns1/ns2.vercel-dns.com) — 사용자가 kimpwatch.kr에서 동일 방식 운영 중(익숙함 우선).
  ⚠️ 이후 Search Console·네이버 TXT 소유확인은 가비아가 아닌 **Vercel 대시보드 DNS Records**에서 추가.
- **TLD 결정**: 지금은 **.com만 구매**. .kr/.co.kr은 SEO 이득 없고(전부 리다이렉트+canonical 단일) 출시 전 선점 리스크 낮음
  → 수익화/브랜드 검색 발생/상표 출원 시점에 방어적 등록.
