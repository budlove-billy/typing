# 기억 축 게임 2종 — 구현 스펙

> 대상: `index.html`(= `brain_app.html`) 단일 HTML SPA "말랑말랑 두뇌체조"
> 목적: 현재 비어 있는 **기억 축**을 채우는 두뇌 트레이닝 게임 2종.
> 원작 근거: NDS 「매일매일 두뇌 트레이닝」의 *순간 기억* / *몇 명일까(인원수 세기)*.
> 원칙: 서버 없음 · 단일 파일 · 탭/클릭/키보드만 사용 · 기존 컴포넌트/패턴 재사용.

---

## 0. 공통 설계 (두 게임 모두 적용 — 원작의 성공 공식)

| 요소 | 방식 |
|---|---|
| **적응형 난이도** | 성공하면 레벨↑, 실패하면 레벨↓(하한 있음). 시니어·아동·성인을 한 게임으로 커버. |
| **생명(lives)** | 3개. 오답 시 −1, 0이 되면 종료. (타이머 대신 실력 기반 종료 → 잘하면 오래 함) |
| **점수 영속화** | `localStorage`. 난이도별 **전체 최고** + **오늘 최고**를 저장·표시 → "매일매일" 재방문 훅. |
| **결과 공유 카드** | 결과 화면에 canvas로 점수 카드를 그려 이미지 저장/공유(2차 확장 훅, 아래 §3). |
| **다국어** | 모든 UI 문자열은 `I18N` 객체에 `{ko,en,th}` 3언어로 등록. 하드코딩 금지. |

### localStorage 스키마
```
brain.flash.best  = { easy:{all:N, day:N, date:"YYYY-MM-DD"}, normal:{…}, hard:{…} }
brain.count.best  = { easy:{all:N, day:N, date:"YYYY-MM-DD"}, normal:{…}, hard:{…} }
```
- 읽기/쓰기는 `try/catch`로 감싸 로컬스토리지 차단 환경에서도 죽지 않게.
- `day.date`가 오늘이 아니면 `day`를 0으로 리셋 후 갱신 → "오늘 최고" 자동 일일 리셋.

## 1. 순간기억 (Flash Memory) — `id: flash`

### 게임성
격자 셀에 숫자 `1..N`이 무작위 위치에 **잠깐 표시**됐다가 사라진다. 플레이어는 빈 셀을
**오름차순(1→2→…→N)** 으로 탭한다. 전부 맞히면 클리어.

### 규칙
- **격자**: 5×5(25칸) 고정. 그 안에 `N`개 숫자를 무작위 배치.
- **표시 시간**: `reveal = clamp(2200 − N×120, 700, 2200)` ms. N이 커질수록 짧아짐.
- **입력**: 숫자를 숨긴 뒤 빈 타일을 순서대로 탭.
  - 맞으면 타일에 숫자 표시 + 초록 고정.
  - **틀리면** 정답 타일을 잠깐 깜빡여 알려주고 라운드 실패.
- **적응형**:
  - 클리어 → `score += N`, `N = min(N+1, 25)`.
  - 실패 → `lives−1`, `N = max(N−1, 3)`.
  - `lives==0` → 종료.
- **레벨 시작값 & 속도**(난이도 3단계, 기존 `.diff-btn` 3열 UI 재사용):

  | 난이도 | 시작 N | 표시시간 배수 |
  |---|---|---|
  | 🌱 쉬움 easy | 3 | ×1.3 |
  | ⚡ 보통 normal | 4 | ×1.0 |
  | 🔥 어려움 hard | 5 | ×0.75 |

### 화면(3 카드: intro / game / result)
- **intro**: 제목·설명·난이도 3버튼·시작 버튼. 하단에 최고기록(오늘/전체) 표시.
- **game**: 상단 SCORE·LIFE(♥♥♥)·LEVEL 스탯 → 5×5 보드 → 상태 문구("기억하세요!" / "순서대로 탭!").
- **result**: 이모지·타이틀(신기록 여부)·최종 점수·도달 최고 레벨·오답·최고기록·[새 게임].

### 상태 객체(초안)
```js
const FM = {
  diff:'normal', n:4, score:0, lives:3, maxLevel:0,
  cells:[],        // 25칸: {num|null}
  expect:1,        // 다음에 탭해야 할 숫자
  phase:'idle',    // idle|show|input|over
  best:{…}         // localStorage 로드
};
```

### 핵심 함수
`setFlashDiff(d)` · `startFlash()` · `fmNewRound()`(배치+표시) · `fmHide()`(숨김→input) ·
`fmTapCell(i)`(채점) · `fmClear()` / `fmFail()` · `finishFlash()` · `resetFlash()`.

## 2. 인원수 세기 (Head Count) — `id: count`

### 게임성 (원작 기믹 보존)
집(🏠)으로 사람이 **들어가면 집에 가려 보이지 않는다**. 나올 때만 다시 보인다.
일련의 입·출 이벤트가 애니메이션으로 지나간 뒤 **"지금 집 안에 몇 명?"** 을 맞힌다.
→ 화면에 사람이 계속 쌓여 보이면 그냥 세면 되니까(암산), **집이 가려서 기억(암산+작업기억)** 이 되게 함.

### 규칙
- 시작 인원 0. `events`개의 이벤트를 순차 재생. 각 이벤트는 `+k`(입장) 또는 `−k`(퇴장).
  - 내부 인원이 **음수가 되지 않게** 생성(퇴장 수 ≤ 현재 인원).
  - `k`는 1~3(난이도에 따라).
- 입장: 사람 스프라이트가 화면 가장자리에서 집으로 이동 → 집 앞에서 **fade-out**(숨음).
- 퇴장: 집에서 **fade-in** → 가장자리로 이동 후 사라짐.
- 재생이 끝나면 4지선다(정답 + 그럴듯한 오답 3개)로 질문. 탭하면 채점.
- **적응형**: 정답 → `score += 난이도점수 + streak보너스`, `streak++`, 난이도 within-tier 살짝 상승(이벤트 수/속도).
  오답 → `lives−1`, `streak=0`. `lives==0` → 종료.
- **난이도 3단계**(시작 파라미터):

  | 난이도 | 이벤트 수 | 이벤트 간격 | k 최대 |
  |---|---|---|---|
  | 🌱 쉬움 easy | 3~4 | 900ms | 1 |
  | ⚡ 보통 normal | 5~6 | 700ms | 2 |
  | 🔥 어려움 hard | 7~9 | 500ms | 3 |

### 화면(3 카드: intro / game / result)
- **intro**: 제목·설명·난이도 3버튼·시작·최고기록.
- **game**: 상단 SCORE·LIFE·STREAK → 무대(stage: 집 + 사람 애니메이션 레이어) →
  재생 후 질문 영역(문구 + 4지선다 버튼).
- **result**: 이모지·타이틀·최종 점수·최고 연속(streak)·최고기록·[새 게임].

### 상태 객체(초안)
```js
const HC = {
  diff:'normal', score:0, lives:3, streak:0, maxStreak:0,
  inside:0, answer:0, events:[], playing:false, best:{…}
};
```

### 핵심 함수
`setCountDiff(d)` · `startCount()` · `hcBuildSequence()`(이벤트 생성, inside≥0 보장) ·
`hcPlay()`(setTimeout 체인 애니메이션) · `hcAskQuestion()`(선택지 생성) ·
`hcAnswer(v)`(채점) · `finishCount()` · `resetCount()`.

### 애니메이션 메모
- 의존성 없이 CSS `transition: transform/opacity`로 스프라이트(🧍/🧑) 이동.
- `setTimeout` 체인으로 이벤트를 직렬 재생. 각 이벤트 종료 시 `inside` 갱신(내부 상태만, 화면엔 숫자 미표시).
- 재생 중 나가기/재시작 대비: 모든 타이머 핸들을 배열에 모아 종료 시 `clearTimeout`.

## 3. 결과 공유 카드 (2차 확장 훅 — 두 게임 공용)
- 결과 카드에 "🔗 결과 이미지" 버튼 추가.
- `<canvas>`에 배경·로고·게임명·점수·최고레벨/연속·날짜를 그려 `toBlob → download` 또는
  Web Share API(`navigator.share`, 모바일)로 공유.
- 두 게임이 같은 `renderShareCard({title,score,sub})` 헬퍼를 공유 → 무료 바이럴 엔진.
- 본 커밋 범위에서는 **훅 지점만 남기고**(함수 stub/주석) 실제 canvas 렌더는 후속 작업으로.

## 4. 인지 영역 커버리지(왜 이 2종인가)
현재 두뇌체조: 한글타자·영단어·IQ·스도쿠·암산 → **계산/타이핑/논리** 축만 존재.
- **순간기억** = 시공간 작업기억(visuo-spatial working memory).
- **인원수 세기** = 청·시각 순차 작업기억 + 갱신(updating).
→ 비어 있던 **기억 축**을 정면으로 채움. 다음 후보는 주의·속도 축(스트룹·순서연결).
