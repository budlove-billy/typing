## 1. 🌈 관찰력 sight — "다른 모양 찾기 (odd)"  ★우선순위 1
**한 줄:** 같은 그림이 깔린 판에서 딱 하나 **회전/거울로 다른** 타일을 찾아 탭.

- **차별점:** 기존 spot은 *명도 차이*(색). odd는 *형태(방향)* 차이 → 관찰 축을 색↔형태로 이분.
- **코어 루프(0–3s):** n×n 격자에 동일 비대칭 글리프(예 ◤/🦶/L자). 1칸만 회전 or 좌우반전. 탭 → 정답 팝·다음 판.
- **세션 루프(60s):** 타임어택. 정답마다 +점수·combo, 오답 −2s(spot 관례). 3판마다 판 확대(4→8 상한), 회전각/거울차 **미세화**로 난도↑.
- **튜닝 레버:** 판크기, 차이유형(회전만/거울만/혼합), 회전각(90°→45°→15° `[PH]`), 제한시간.
- **생성/검증:** "정답 1개만 존재" 불변식 — 나머지 전 타일 동일 변환, 정답만 다른 변환. 시뮬로 유일성 확인.
- **연동:** `ABILITY_MAP.odd={focus:40,space:30,speed:30}` REF `[PH]350`. 미션 풀 B.
- **비주얼:** 비대칭 글리프 필요(대칭 이모지 금지 — 회전/거울 구분 불가). 마스코트 실루엣 활용 검토.

## 2. 🔨 협응력 coord — "마시멜로 받기 (catch)"  ★우선순위 2
**한 줄:** 떨어지는 마시멜로를 좌우로 움직여 받고 포크(🍴)는 피한다.

- **차별점:** whack은 *제자리 두더지 탭*. catch는 *연속 이동+타이밍* → 협응 축을 반응↔추적으로 이분.
- **코어 루프(0–2s):** 하단 마스코트 바구니, 화면 상단서 아이템 낙하. 좌/우 버튼(+키보드/드래그)로 이동. 마시멜로 받으면 +1·팝, 포크 받으면 lives−1(또는 시간−). 놓친 마시멜로 = combo 리셋.
- **세션 루프:** lives 3 or 60s. 낙하 속도·동시 개수·포크 비율이 점수 따라 가속(chop의 accel 선례).
- **튜닝 레버:** 낙하속도, 스폰 간격, 포크 확률, 동시 낙하 수, 바구니 이동속도 `[PH]`.
- **공정성:** 동시 낙하 시 **인접 두 레인에 마시멜로/포크 동시 배치 금지**(피할 수 없는 상황 차단) — 스폰 규칙 시뮬 검증.
- **연동:** `catch={speed:50,focus:30,space:20}` REF `[PH]300`. 미션 풀 B. 마스코트 활용 2호.

## 3. 🎵 청각 sound — "리듬 따라하기 (rhythm)"  ★우선순위 3
**한 줄:** 들려준 **박자 패턴**을 그대로 탭으로 재현.

- **차별점:** melody는 *음높이 순서(사이먼)*. rhythm은 *타이밍/간격* → 청각 축을 음정↔리듬으로 이분.
- **코어 루프:** 단일 패드(또는 2패드)로 짧은 리듬(예 ♩♩♫♩) 재생 → 사용자가 간격 맞춰 탭. 각 탭 타이밍 허용오차 내면 정답.
- **세션 루프:** 성공 시 패턴 길이+1(melody 관례), 실패 lives−1(3). 최장 길이 기록.
- **튜닝 레버:** 허용오차(±ms `[PH]`), 시작 길이, BPM, 패드 수(1→2→4).
- **기술:** 기존 WebAudio `_beep` 재사용, 타이밍은 `performance.now()`(react 선례). **자동재생 정책** 주의 — 첫 탭 후 오디오 컨텍스트 resume.
- **접근성:** 소리 못 듣는 환경 대비 **시각 플래시 동반**(패드 발광) — 무음도 플레이 가능하게.
- **연동:** `rhythm={memory:50,focus:30,speed:20}` REF `[PH]`(최장 길이 기반). 미션 풀 A.

## 4. 📚 언어감각 lang — "글자 맞추기 / 애너그램 (anagram)"  ★확정: vocab/typing 대체 게임
> **결정(2026-07-11 사용자):** 언어 축은 vocab/typing을 되살리지 않고 **신규 게임으로 대체**.
> vocab/typing **소스코드는 삭제하지 않고 보존**(현행 주석 비활성 유지). anagram이 lang 축 능력치를 담당.

**한 줄:** 뒤섞인 글자 타일을 재배열해 원래 단어를 복원. 60초 타임어택.

### 왜 애너그램인가 (vocab/typing이 죽은 이유를 정면 해결)
- vocab/typing 폐기 사유 = **ko 전용 → en/th 불가** + 브레인게임 포지셔닝 불일치.
- 애너그램은 **구조가 언어 독립적**: ko=음절 타일 재배열, en=알파벳 타일 재배열 → **동일 엔진**으로 ko+en 동시 출시 가능(vocab/typing보다 진일보).
- **정답을 미리 알고 맞추는 방식** → 임의 입력을 사전으로 검증할 필요 없음 → **오픈 사전 불필요·욕설 리스크 0**(큐레이션 단어 리스트만).
- **콘텐츠 파이프라인 불필요**: 정적 단어 리스트에서 무한 셔플. moamoa처럼 D+25 재제작 부담 없음.

### 코어 루프 (0–10s)
- 상단 빈 슬롯 N개(글자 수만큼), 하단에 셔플된 타일 N개.
- 타일 탭 → 다음 빈 슬롯 채움. 채운 슬롯 탭 → 타일 회수(오배치 수정).
- 슬롯 전부 채우면 **자동 판정**: 조합 문자열 == 원단어면 정답(팝·+점수·다음 단어), 아니면 흔들림+타일 복귀.
- **힌트 3회**: 아직 안 놓인 정답 첫 글자를 해당 슬롯에 고정.

### 세션 루프 (60s)
- 60초간 최다 단어. 점수 = base(글자 수 비례) + 콤보 + 빠른 해결 보너스.
- 난이도 = 글자 수: easy 3–4 / normal 4–5 / hard 5–7 타일.
- **오답 시 시간 페널티 없음, 콤보만 리셋**(bubble 선례). 지식형이라 과한 처벌은 이탈 유발 → 관대하게.

### 다국어 전략 (핵심 차별)
- **출시 = ko + en 동시**(음절 타일 vs 알파벳 타일, 로직 공유). ← vocab/typing 대비 실질 개선점.
- **th = 보류**: Thai 자모 결합이 복잡 → 타일화 난이도. th 로케일에선 게임 숨김(moamoa koOnly 필터 패턴 응용, 단 anagram은 `koEnOnly`류). 리스트 준비되면 확장.
- 단어 리스트: `WORDS={ko:{3:[..],4:[..],5:[..]}, en:{...}}` 버킷. ko 시드는 **moamoa 낱말 자산 재활용** 검토. en은 가족친화 상용어 큐레이션(~300–500).

### 생성/검증 불변식
- 난이도 버킷에서 무작위 단어 → 타일 셔플. **셔플 결과 문자열 == 원단어면 재셔플**(자명한 판 방지).
- **중복 글자 처리**: 판정은 타일 정체성이 아니라 **조합 문자열 == 원단어**로 비교 → "사과"·"APPLE"의 중복 글자 자동 처리(중복 순열도 정답 인정).
- 시뮬: 각 리스트 전 단어에 대해 (a)셔플≠원단어 보장, (b)정답 조립 가능 확인.

### 튜닝 레버
글자 수(난이도), 제한 60s, 힌트 3, 콤보 보너스 곡선, base 점수 계수 — 전부 `[PH]`.

### 연동 (§12 체크리스트)
- `ABILITY_MAP.anagram={lang:70, memory:20, focus:10}` / `GAME_REF.anagram [PH]`(60초 해결 수 기반).
- grp `cat.lang`, emoji `🔤`(nback 🔡와 구분), best `brain.anagram.best`.
- **braintype 언어가형 추천**을 현재 vocab→nback에서 **→anagram**으로 갱신(원래 언어 유형이 언어 게임으로 연결되게 정상화).
- **미션 풀 제외**(A/B/C는 기억/주의·유연 구성 — lang 부적합, calc 관례처럼 제외).
- HOME_GAMES 노출 필터: ko/en에서 표시, th 숨김.

### 대안(채택 안 함, 기록용)
- **초성 퀴즈**(ㅁㄹㅁㄹ→말랑말랑): 한국 바이럴 강함이나 **ko 전용**(en/th 무대응) → 대체 취지에 안 맞아 기각.
- **숨은 단어 찾기(word-search)**: 관찰+언어 하이브리드지만 사전 부담 유사·재미 밀도 낮아 후순위.

## 5. 🔢 논리력 logic — "픽셀 로직 / 노노그램 (nono)"  ★우선순위 5
**한 줄:** 행·열 숫자 힌트로 칠할 칸을 추론해 픽셀 그림 완성.

- **차별점:** sudoku(제약충족)·sort(정렬)와 다른 *연역 추론* 결. 완성 시 그림 공개 = 만족감 훅.
- **코어 루프:** 셀 탭=채움/×/빈칸 토글. 행·열 힌트 충족 검사.
- **세션 루프:** 5×5/10×10/15×15 난이도. 점수=base−시간−오답. 완성 그림 공유(공유카드에 완성 픽셀).
- **생성/검증:** 그림→힌트 생성은 자명. **유일해(logically solvable without guessing) 보증**이 관건 — 유일성 솔버로 판 필터(sort 솔버 선례). 초기엔 큐레이션 5×5 팩으로 리스크 축소.
- **튜닝 레버:** 판 크기, 채움 비율, 오답 페널티.
- **연동:** `nono={logic:60,focus:25,space:15}` REF `[PH]`. 미션 풀 제외(논리 축 관례, sort/sudoku처럼).
- **비용 경고:** 유일성 솔버가 무거움 → **빌드 비용 상(上)**. 우선순위 뒤로.

## 6. 🧊 공간감각 space — "블록 채우기 (fit)"  ★우선순위 6
**한 줄:** 폴리오미노 조각 3개를 격자에 배치, 줄을 채워 지우는 우드쿠류.

- **차별점:** rotate(심적 회전)·slide(순차 이동)와 다른 *공간 배치/계획* 결. 리텐션 강한 캐주얼 포맷.
- **코어 루프:** 하단 3조각 → 드래그로 격자 배치, 못 놓으면 게임오버. 행/열 꽉 차면 소거+점수.
- **세션 루프:** 소거 지속=고점. 남은 공간 계획이 핵심 결정.
- **공정성/검증:** "3조각 다 못 놓으면 종료"라 **데드락 즉시성** 균형이 관건 — 조각 풀 가중치로 과도한 조기 종료 방지. 시뮬로 평균 생존 판수 측정.
- **튜닝 레버:** 격자 크기(8×8 `[PH]`), 조각 세트(회전 허용?), 소거 콤보 보너스.
- **연동:** `fit={space:60,logic:25,focus:15}` REF `[PH]`. 미션 풀 C.

## 7. 🎯 집중력 focus — "화살표 집중 / 플랭커 (flank)"  ★우선순위 7
**한 줄:** 가운데 화살표 방향만 답하고 양옆 방해 화살표는 무시.

- **차별점:** stroop(색-단어 간섭)·switch(규칙 전환)와 다른 *공간적 방해 억제(flanker)* — 인지심리 정석 과제.
- **코어 루프:** `<< < > <<` 형태. 가운데(`<`) 방향 버튼 탭. 일치/불일치 시행 혼합.
- **세션 루프:** 60s 타임어택, combo, 오답 −2s(축 관례). 불일치 비율↑로 난도.
- **튜닝 레버:** 불일치 확률, 플랭커 개수, 표시 시간.
- **연동:** `flank={focus:60,speed:30,logic:10}` REF `[PH]`. 미션 풀 B.

## 8. ➗ 계산감각 calc — "어림 계산 (guess)"  ★우선순위 8 (백로그에 이미 있음)
**한 줄:** 식의 **근사값**을 3택으로 빠르게. 정밀계산 아님.

- **차별점:** math(정확 암산)·bubble/merge와 다른 *수 감각/추정*. MEMORY '다음 후보'에 기재됨.
- **코어 루프:** "47×8 ≈ ?" → [320/380/500] 중 근사 정답 탭. 60s.
- **세션 루프:** combo, 오답 −2s. 자릿수·연산 복잡도↑.
- **생성/검증:** 오답 보기는 정답과 **판별 가능한 간격**(너무 가까우면 추정 무의미) — 보기 생성 규칙 시뮬.
- **연동:** `guess={calc:70,speed:20,focus:10}` REF `[PH]`. 미션 풀 제외(계산 축 관례).

## 9. 🧠 기억력 memory — "거꾸로 기억 (rev)"  ★우선순위 9 (선택)
**한 줄:** 하나씩 나타난 숫자열을 **역순**으로 입력(backward digit span).

- **차별점:** flash(격자 순서)·nback(N칸 전)·cards(위치)와 다른 *작업기억 조작(역순)*. 기억 축 이미 두터워 후순위.
- **코어 루프:** 3→4→5…자리 순차 표시 후 키패드로 거꾸로 입력. 성공 길이+1, 실패 종료.
- **연동:** `rev={memory:70,focus:30}` REF `[PH]`. 미션 풀 A(넣으면 A 과밀 주의).

## 11. 배포 로드맵 (라이브 안전 우선)

**Wave 1 — 얇은 축 메우기(빠른 승리):** odd(관찰) → catch(협응) → rhythm(청각)
  이유: 각 축 1종뿐, 신규 1종의 능력치/다양성 기여가 가장 큼. 생성 로직 단순 = 리스크 낮음.
**Wave 2 — 신규 포맷 리텐션:** fit(블록채우기, 캐주얼 훅) → flank(집중)
**Wave 3 — 콘텐츠/판단 필요:** anagram(언어, ko+en 단어리스트 준비 필요) → nono(논리, 솔버 비용) → guess(계산) → rev(기억)

각 Wave는 **1종씩 배포 후 안정 확인** 다음 진행. 절대 일괄 배포 금지.
